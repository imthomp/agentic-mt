# Phase 1: QE-vs-human-judgment for low-resource pairs

## What this is

A meta-evaluation of reference-free QE (COMET-KIWI) against real human
judgment for low-resource language pairs, using `src/agentic_mt/qe/`.

## Data note (important — read before reusing this setup)

FLORES-200 does **not** carry human MQM/DA quality judgments — it's a
parallel-text corpus, not a human-evaluation dataset. Human judgments for
this kind of analysis live in the WMT Metrics shared-task collections. We
use the real WMT Direct Assessment (DA) human scores from
`RicardoRei/wmt-da-human-evaluation` (HuggingFace), which pairs existing
WMT-submitted system translations (mostly open-source MT systems, not
translations generated by us) with real human DA scores.

## Reuse of multilingual-mqm-benchmark

Rather than re-run COMET-KIWI/COMET/chrF ourselves, we join freshly-loaded
human DA scores with metric scores already computed in
`projects/multilingual-mqm-benchmark/multilingual-mqm-benchmark/results/`.
Those checkpoint CSVs don't carry a unique segment key, so the join relies
on order-preserving `lang`-column filtering (see `qe/data.py` docstring) —
verified by checking per-language row counts match across sources, and by
confirming our recomputed Pearson/Spearman values match mqmbench's
independently-computed `correlations.csv` to ~10 significant figures.

## Results (run 2026-07-06T19-12-20)

COMET-KIWI (reference-free QE) vs. human DA score, Spearman r with 95%
bootstrap CI (1000 resamples):

| Pair | Spearman r | 95% CI | n | Flagged (<0.3)? |
|---|---|---|---|---|
| ha | 0.115 | [0.102, 0.127] | 23,983 | **yes** |
| km | 0.457 | [0.434, 0.480] | 4,722 | no |
| ps | 0.295 | [0.269, 0.321] | 4,611 | **yes** |
| xh | 0.342 | [0.307, 0.378] | 2,502 | no |
| zu | 0.247 | [0.212, 0.283] | 2,968 | **yes** |

Reference-based COMET and chrF are consistently a bit higher than
COMET-KIWI but track the same pattern (full breakdown in each run's
`summary.csv`).

## Interpretation

3 of 5 pairs (ha, ps, zu) fall below the 0.3 correlation threshold. This is
a genuine limitation, not a bug to route around: for these pairs,
QE-in-the-loop refinement (using COMET-KIWI to decide when to
retry/refine a translation) is not currently viable without QE adaptation
for the target language. Report this as a limitation. Khmer and Xhosa are
closer to usable but still weak by high-resource-pair standards (e.g. en-de
COMET-KIWI Spearman r ≈ 0.57 in mqmbench's results).

xh/zu also show a visibly banded/quantized scatter pattern (see each run's
`scatter.png`), consistent with a small annotator pool — worth flagging if
this goes into a paper.
